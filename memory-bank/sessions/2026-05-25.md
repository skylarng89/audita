### Tenant resolution bug (critical)
- **Symptom:** After admin setup, login returns "Invalid credentials" / 403 "Unknown tenant."
- **Root cause:** Frontend resolves subdomain `cm` from `cm.mypixelpay.com` and sends `X-Tenant-Slug: cm`, but the tenant slug created during setup was `pixelpay-systems-limited`. The `TenantResolutionFilter` rejects `cm` as unknown.
- **Fix:** Implemented ADR-022 — subdomain-based tenant resolution using `X-Forwarded-Host` header.

### Changes made (ADR-022)
- **V2 Flyway migration:** Adds nullable `subdomain` column to `tenants` table with unique partial index.
- **TenantEntity:** Added `subdomain` field, getter/setter, 3-arg constructor.

- **TenantResolutionFilter (complete rewrite):** Resolution order: (1) `X-Forwarded-Host` subdomain → DB lookup, (2) `X-Tenant-Slug` header → DB lookup. Reserved subdomains (`www`, `app`, `api`, `mail`, `smtp`) are skipped.
