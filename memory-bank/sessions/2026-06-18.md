## Changes

### RBAC Permission Enforcement
- Replaced all 88 `@PreAuthorize` annotations across 16 controllers from `hasRole()`/`hasAnyRole()` to `@authz.hasPermission(authentication, 'permission.code')`
- Created `AuthorizationLogic` bean (`@Component("authz")`) with `hasPermission`/`hasAnyPermission` methods that check for `*` wildcard (SUPER_ADMIN) or exact permission match
- Added `SimpleGrantedAuthority("*")` to `UserPrincipal.ofSuperAdmin()` for wildcard access
- Removed `Approver` system role — migrated to 3 system roles (Admin, Requester, Auditor)
- Added 4 new permissions: `cr.view.all`, `cr.manage_participants`, `uat.signoff`, `deployment.execute` (22 total)

## Decisions Made
- Permission-based RBAC using existing infrastructure (permissions table, JWT claims, UserPrincipal authorities were already provisioned but unenforced)
- SUPER_ADMIN gets `*` wildcard authority — bypasses all permission checks
- Watchers can comment + receive notifications but cannot approve/reject
- Deployment model: single assignee (replaced multi-approver), deployer marks completion, requester closes CR
- All approvers are required (no optional) — simpler evaluation logic
- Read-only boundary: `completionStatus == COMPLETED` blocks all mutations including comments
- Core fields locked after submission (DRAFT-only editing)
- Self-approval blocked for non-admin creators (existing behavior preserved)
- UAT requester sign-off remains (ADR-045)
- Custom role overlap: no full-permission roles, no exact-set duplicates, partial overlap allowed
