## 2. Groups Page ‚Äî Fix Silent Error Swallowing

### Problem
Groups not rendering on `/groups` page despite API returning valid `Page<Group>` response. Root cause: `useAsyncData` callback used `.catch(() => null)`, silently swallowing any errors.

### Changes

**`audita-web/pages/groups/index.vue:233-245`** ‚Äî replaced:
```typescript
() => fetchGroups(page.value, pageSize).catch(() => null)
```

With explicit async error handling:
```typescript
async () => {
  loadError.value = ""
  try {
    return await fetchGroups(page.value, pageSize)
  } catch (err: unknown) {
    loadError.value = resolveApiErrorMessage(err, "Failed to load groups.")
    return { content: [], totalElements: 0, totalPages: 0, size: pageSize, number: 0 }
  }
}
```

Also removed `onMounted(() => { refresh() })` workaround ‚Äî `watch: [page]` already handles re-fetch on page change.

### ADRs
- ADR-032: useAsyncData Error Handling ‚Äî Explicit try/catch Over Silent .catch(() => null)
