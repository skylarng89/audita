### Bug Fix: Missing `user_roles` Table for Existing Tenants

**Problem**: `SlaMonitoringService` threw `PSQLException: ERROR: relation "user_roles" does not exist` for tenant `ronin-limited` because the `V6__add_user_roles.sql` migration was never applied to existing tenants.

**Root Cause**: `FlywayTenantMigrator.migrate()` was only called during tenant provisioning (`TenantService.provision()`) and first-run setup (`TenantService.setupSingleTenant()`). There was no startup component to apply pending migrations to pre-existing tenants.

**Fix**: Added `TenantMigrationStartupRunner` — an `@EventListener(ApplicationReadyEvent.class)` component that iterates all existing tenants via `TenantRepository.findAll()` and runs `flywayTenantMigrator.migrate(slug)` for each on application startup. Failures are logged but do not block startup (other tenants continue migrating).

**Files Changed**:
- `audita-api/infrastructure/src/main/java/io/audita/infrastructure/tenant/TenantMigrationStartupRunner.java` — new file
- `audita-api/infrastructure/src/main/java/io/audita/infrastructure/tenant/FlywayTenantMigrator.java` — updated Javadoc to reference `TenantMigrationStartupRunner`
