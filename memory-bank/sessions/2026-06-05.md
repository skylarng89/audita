## Integration Fixes

During verification, several cross-phase issues were discovered and fixed:

1. **Missing `token_version` column** — Created `V5__add_token_version.sql` migration (SA14-005 added entity field but no DB migration)
2. **JWT filter token version check ordering** — `userRepository.findById()` was called BEFORE `resolvePrincipal` set `TenantContext`, causing cross-schema query failures. Moved check after tenant context initialization.
3. **IdempotencyKeyRepository JPQL CAST UUID** — `CAST(... AS UUID)` fails in JPQL. Converted `updateResourceId()` from JPQL to native SQL.
4. **AuthServiceTest mock setup** — Added `@Mock RateLimitService` and replaced `any()` with `anyInt()` for the primitive `int tokenVersion` parameter in `jwtService.issue()`.
5. **SecurityConfig double-modification** — Phase B and D both modified SecurityConfig; reconciled to single config with CSRF disabled + security headers applied.
